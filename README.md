# 日/月计划和成果验收系统

> 一套面向部门内部的工作计划管理与成果验收系统，覆盖从「制定计划 → 审批 → 执行 → 成果验收」的完整闭环。

---

## 一、项目简介

本系统为部门/团队（约 10-30 人）提供日常工作计划管理和成果验收功能。员工可以便捷地填写**日报、周报、月报**，提交后由领导**逐级审批**；计划执行完毕后，员工提交**成果报告**供领导验收。系统支持日报→周报→月报的逐级汇总引用，方便阶段性回顾总结。

### 核心流程

```
员工填写计划（日/周/月）→ 提交审批 → 领导多级审批（通过/驳回/转审）
       │                                       │
       │                               ┌──────┴──────┐
       │                               ▼             ▼
       │                          审批通过        审批驳回
       │                               │       （修改后重新提交）
       ▼                               ▼
   员工执行计划                 员工提交成果
                                    │
                                    ▼
                              领导验收成果
                              ┌────┴────┐
                              ▼         ▼
                          验收通过    验收驳回
                                    （修改后重新提交）
```

## 二、用户角色

| 角色 | 说明 | 核心权限 |
|------|------|----------|
| **员工** | 普通团队成员 | 填写个人计划和成果、查看自己的计划和成果 |

## 三、功能清单

### 3.1 计划管理
- ✅ 日/周/月三级计划体系
- ✅ 计划 CRUD（创建、编辑、查看、删除）
- ✅ 计划状态流转：草稿 → 已提交 → 审批中 → 已通过/已驳回
- ✅ 多维度筛选：类型、状态、优先级、时间范围、分类、关键词
- ✅ 日历视图（日/周/月切换，不同颜色标识状态）
- ✅ 日报→周报→月报汇总引用

### 3.2 成果管理
- ✅ 成果 CRUD，与计划一一关联
- ✅ 计划 vs 实际量化对比（指标、耗时偏差一目了然）
- ✅ 成果状态流转：待填写 → 已提交 → 验收中 → 已通过/已驳回
- ✅ 附件上传（图片、文档等成果佐证材料）

### 3.3 审批流程
- ✅ 多级审批链（默认两级：直属领导 → 部门负责人）
- ✅ 审批操作：通过 / 驳回（附意见）/ 转审（同部门内）
- ✅ 同级多审批人：全部通过后进入下一级
- ✅ 同级审批人可互相查看审批意见
- ✅ 审批时间线展示完整审批轨迹

### 3.4 统计报表
- ✅ 个人统计：完成率、按时提交率、审批通过率
- ✅ 团队统计（领导可见）：整体完成率、个人排名、未提交人员
- ✅ 可视化图表：饼图（状态分布）、柱状图（完成率对比）、折线图（趋势）

### 3.5 消息通知
- ✅ WebSocket 实时推送
- ✅ 通知铃铛 + 未读红点 + 通知列表
- ✅ 通知场景：新审批到达、审批结果（通过/驳回）、超时提醒

### 3.6 系统管理
- ✅ 用户管理（添加/编辑/禁用/重置密码）
- ✅ 部门管理（树形结构）
- ✅ 审批链配置（按部门+计划类型配置审批级数和审批人）
- ✅ 项目分类管理

## 六、快速开始

### 6.1 前置要求

- Docker & Docker Compose（推荐）
- 或手动安装：Java 17、Node.js 18+、MySQL 8.0、Redis 7

### 6.2 Docker 一键部署（推荐）

```bash
# 1. 克隆仓库
git clone https://github.com/fsing111/---.git
cd 日-月计划和成果验收系统

# 2. 启动所有服务
chmod +x start.sh stop.sh
./start.sh
```

启动后访问：
- **前端页面**：http://localhost
- **后端 API 文档**：http://localhost:8080/doc.html
- **默认管理员账号**：`admin` / `admin123`

### 6.3 手动开发部署

#### 后端

```bash
cd backend
mvn clean install
mvn spring-boot:run -Dspring-boot.run.profiles=dev
```

#### 前端

```bash
cd frontend
npm install
npm run dev
```

### 6.4 停止服务

```bash
./stop.sh
# 或
docker compose down
```

## 八、核心业务规则

| # | 规则 | 说明 |
|---|------|------|
| 1 | **默认审批链** | 未配置审批链时，自动回退：直属领导(level=1) → 部门负责人(level=2) |
| 2 | **同级全部通过** | 同一级配置多个审批人时，全部通过后才进入下一级 |
| 3 | **同级互见** | 同级审批人可查看彼此的审批意见 |
| 4 | **驳回从头来** | 被驳回后修改重新提交，从第一级审批重新开始 |
| 5 | **转审限同部门** | 审批只能转给同部门的人 |
| 6 | **数据权限隔离** | 员工只看到自己的数据，领导看到下属的数据（递归），管理员看全部 |
| 7 | **计划与成果一对一** | 每条计划最多关联一个成果 |
| 8 | **成果关联约束** | 只有已通过的计划才能提交成果 |
| 9 | **文件限制** | 单文件 ≤ 10MB，单次 ≤ 5 个附件 |
| 10 | **超时不阻止** | 计划超时提交不会被系统阻止，仅通过提醒功能督促 |

## 十、API 接口概览

Base URL：`/api/v1`

| 模块 | 接口前缀 | 主要功能 |
|------|----------|----------|
| 认证 | `/auth` | 登录、登出、获取当前用户 |
| 计划 | `/plans` | CRUD、提交、日历视图、汇总引用 |
| 成果 | `/achievements` | CRUD、提交、计划 vs 实际对比 |
| 审批 | `/approvals` | 待审批列表、审批历史、通过/驳回/转审、审批时间线 |
| 通知 | `/notifications` | 通知列表、未读数、标记已读 |
| 文件 | `/files` | 上传、下载、预览 |
| 统计 | `/statistics` | 个人统计、团队统计、趋势 |
| 管理 | `/admin` | 用户/部门/审批链/分类 CRUD |

所有接口返回统一格式：

```json
{
  "code": 200,
  "message": "success",
  "data": {}
}
```
